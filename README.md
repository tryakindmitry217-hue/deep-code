REST-сервис на FastAPI, который принимает с устройств JSON-статистику вида {"x": float, "y": float, "z": float}, сохраняет данные в PostgreSQL с меткой времени и идентификатором устройства, а также поддерживает ведение пользователей и назначение устройств владельцам. Аналитические задачи (расчёт минимума, максимума, количества, суммы и медианы по каждой оси) вынесены в асинхронную очередь Celery на базе Redis, что позволяет получать агрегированную и поустройственную аналитику как за заданный период, так и за всё время по устройству или по всем устройствам пользователя. Всё окружение, включая API, воркер, базу данных и Redis, упаковано в Docker Compose. Для нагрузочного тестирования используется Locust (также запускаемый через Docker Compose).  

Файлы в папке app реализуют всю логику сервиса: 

main.py — точка входа FastAPI: содержит эндпоинты (сбор данных, запрос анализа, получение результата, управление пользователями).  

models.py — модели SQLAlchemy: таблицы users, devices, data_records и связи между ними.  

schemas.py — Pydantic-схемы для валидации входящих запросов и форматирования ответов.  

database.py — настройка асинхронного подключения к PostgreSQL, создание сессий и декларативная база.  

celery_app.py — конфигурация Celery: брокер и бэкенд Redis, сериализация JSON.  

tasks.py — асинхронные задачи Celery (analyze_device, analyze_user) для вычисления статистик (min, max, median и др.) по устройствам и пользователям.
